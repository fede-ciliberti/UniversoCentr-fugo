# AGENTS.md (code)

This file provides guidance to agents when working with code in this repository.

## Non-Obvious Project Structure & Commands

- **Command Execution Path Mismatch**: The main orchestrator script, `computational_implementation/simulations/run_complete_simulation.py`, incorrectly references scripts in a non-existent `notebooks/` directory. The actual paths are within `computational_implementation/`. This must be corrected locally or accounted for during execution.
- **Implicit Data Dependencies**: Analysis scripts like `experimental_validation/hubble_verification/verify_hubble_law_final.py` depend on `.npz` files (e.g., `simulation_results.npz`) being present in the root directory. These are generated by simulation scripts and are not explicitly managed. Ensure simulation scripts are run first to generate these artifacts.
- **Primary Execution Scripts**:
  - **Full Workflow**: `python computational_implementation/simulations/run_complete_simulation.py`
  - **Hubble Verification**: `python experimental_validation/hubble_verification/verify_hubble_law_final.py`
  - **System Test**: `python computational_implementation/analysis_tools/test_validation_system.py`

## Testing & Validation

- **Ad-Hoc Testing**: The project does not use a formal testing framework like `pytest`. Tests are standalone executable scripts (e.g., `test_validation_system.py`) that perform checks and print results to standard output. To run tests, execute the script directly with Python.
- **Multiple Data Formats**: The analysis script `verify_hubble_law_final.py` is designed to handle multiple internal data structures within `.npz` files (`metric_invariants` vs. `metric_evolution`). This is a non-standard feature and key to understanding the analysis logic.

## Code Style & Conventions

- **No Formal Linter/Formatter**: There are no configuration files for tools like Black, Flake8, or Prettier. Code style is inconsistent. Follow the style of the file you are editing.
